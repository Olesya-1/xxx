    14. Практическое занятие 3 Протоколы HTTP и HTTPS. Структура HTTP-запроса и ответа.

    Часть 1: HTTP и HTTPS — основы

    HTTP (HyperText Transfer Protocol) — это протокол прикладного уровня, набор правил, который определяет, как клиент (браузер) формирует запрос к серверу и как сервер формирует ответ.

    HTTPS (HTTP Secure) — это защищенная версия HTTP. Вся коммуникация между клиентом и сервером шифруется с помощью протоколов SSL/TLS. Это защищает данные от перехвата и подмены.

    Часть 2: Структура HTTP-запроса (HTTP Request)

    1 Стартовая строка (Request Line)

    Метод (HTTP Method) — действие, которое мы хотим выполнить с ресурсом.

    2 Заголовки (Headers)

    Это мета-информация о запросе. Набор пар «Ключ: Значение».

    3 Тело запроса (Body)

    Необязательная часть. Используется в методах POST, PUT, PATCH для передачи данных на сервер (например, JSON с информацией о пользователе, XML, файлы).

    Часть 3: Структура HTTP-ответа (HTTP Response)

    Сервер, получив запрос, обрабатывает его и возвращает ответ. Структура ответа похожа на запрос.

    1 Статусная строка (Status Line)

    2 Заголовки ответа (Headers)

    Мета-информация от сервера.

    3 Тело ответа (Body)

    Основные данные, которые запросил клиент. Это может быть HTML-страница, JSON, XML, картинка, CSS-файл и т.д.
