Тест-кейсы. Статусы кейс-тестов и тест-сьюты.

1. Что такое Тест-кейс (Test Case)?

Тест-кейс — это минимальная единица тестирования, представляющая собой formalized описание действий, условий и входных данных, необходимых для проверки конкретной функции или характеристики тестируемого объекта, а также ожидаемого результата.

2. Структура тест-кейса (Атрибуты)

Стандартный тест-кейс содержит следующие поля

  • ID / Уникальный идентификатор
  • Название / Заголовок (Title)
  • Предусловия (Preconditions)
  • Шаги (Steps)
  • Ожидаемый результат (Expected Result)
  • Постусловия (Postconditions)
  • Приоритет (Priority)
  • Серьезность (Severity)

3. Статусы выполнения тест-кейса

В процессе тестирования тест-кейс может иметь следующие статусы:

  • Passed / PASS
  • Failed / FAIL
  • Blocked / BLOCKED
  • Skipped / SKIPPED
  • Not Run / NOT EXECUTED

4. Что такое Тест-сьют (Test Suite)?

Тест-сьют (Набор тестов) — это совокупность тест-кейсов, объединенных общей логикой или целью.

Цели создания тест-сьютов:

  • Организация: Группировка кейсов по функциональным модулям (например, "Тесты для корзины покупок", "Тесты для личного кабинета").
  • Выполнение по цели: Создание наборов для конкретного типа тестирования.
  • Автоматизация: Группировка кейсов для автоматического запуска (например, "API-тесты", "UI-тесты главной страницы").

5. Жизненный цикл тест-кейса

  • Создание (Draft): Аналитик/тестировщик создает кейс на основе требований, используя техники тест-дизайна.
  • Ревью (Review): Кейс проверяют коллеги (другой тестировщик, разработчик, аналитик) на предмет корректности, полноты и отсутствия ошибок.
  • Доработка (Rework): Автор вносит правки по комментариям ревьюера.
  • Утверждение (Approved): Кейс готов к использованию.
  • Выполнение (Execution): Тестировщик запускает кейс и присваивает ему один из статусов (Pass, Fail, etc.).
  • Обновление/Снятие с эксплуатации (Update/Retire): Если функционал изменился, кейс необходимо обновить. Если функционал удален — кейс архивируется.
